## Overview

Dub requires various environment variables for configuration. This page provides a comprehensive reference for all available options.

<Note>
  All environment variables should be set in the `.env` file in the `apps/web` directory.
</Note>

## Required Variables

These variables are essential for Dub to function properly.

### Application Configuration

<ParamField path="NEXT_PUBLIC_APP_NAME" type="string" required>
  The name of your application.
  
  **Default:** `Dub`
  
  ```bash
  NEXT_PUBLIC_APP_NAME=Dub
  ```
</ParamField>

<ParamField path="NEXT_PUBLIC_APP_DOMAIN" type="string" required>
  The primary domain where your Dub instance is hosted.
  
  **Example:** `dub.co`
  
  ```bash
  NEXT_PUBLIC_APP_DOMAIN=yourdomain.com
  ```
</ParamField>

<ParamField path="NEXT_PUBLIC_APP_SHORT_DOMAIN" type="string" required>
  The domain used for shortened links.
  
  **Example:** `dub.sh`
  
  ```bash
  NEXT_PUBLIC_APP_SHORT_DOMAIN=yourdomain.link
  ```
</ParamField>

### Analytics & Tracking

<ParamField path="TINYBIRD_API_KEY" type="string" required>
  Your Tinybird API authentication token. Required for analytics functionality.
  
  Get your token at: [Tinybird Auth Tokens](https://www.tinybird.co/docs/concepts/auth-tokens.html)
  
  ```bash
  TINYBIRD_API_KEY=p.ey...
  ```
</ParamField>

<ParamField path="TINYBIRD_API_URL" type="string" required>
  The Tinybird API endpoint for your region.
  
  **Default:** `https://api.tinybird.co`
  
  **Regions:**
  - US East: `https://api.tinybird.co`
  - EU: `https://api.eu-central-1.tinybird.co`
  - US West: `https://api.us-west-2.tinybird.co`
  
  [View all regions](https://www.tinybird.co/docs/api-reference/api-reference.html#regions-and-endpoints)
  
  ```bash
  TINYBIRD_API_URL=https://api.tinybird.co
  ```
</ParamField>

### Database

<ParamField path="DATABASE_URL" type="string" required>
  MySQL database connection URL.
  
  **Format:** `mysql://username:password@host:port/database`
  
  Get started with PlanetScale: [Connect Node.js App](https://planetscale.com/docs/tutorials/connect-nodejs-app)
  
  ```bash
  DATABASE_URL="mysql://user:password@host:3306/dub"
  ```
</ParamField>

<ParamField path="PLANETSCALE_DATABASE_URL" type="string">
  PlanetScale HTTP proxy URL for local development.
  
  **Local development:**
  ```bash
  PLANETSCALE_DATABASE_URL="http://root:unused@localhost:3900/planetscale"
  ```
</ParamField>

### Authentication

<ParamField path="NEXTAUTH_SECRET" type="string" required>
  Secret key for NextAuth.js session encryption. Must be a random 32+ character string.
  
  Generate one at: [generate-secret.vercel.app/32](https://generate-secret.vercel.app/32)
  
  ```bash
  NEXTAUTH_SECRET=your-super-secret-key-here-32-chars-minimum
  ```
</ParamField>

<ParamField path="NEXTAUTH_URL" type="string">
  The canonical URL of your site. Required for localhost development.
  
  **Production:** Auto-detected by Vercel
  
  **Local development:**
  ```bash
  NEXTAUTH_URL=http://localhost:8888
  ```
</ParamField>

### Email Configuration

<ParamField path="RESEND_API_KEY" type="string" required>
  Resend API key for sending emails (production).
  
  Get your API key: [Resend API Keys](https://resend.com/api-keys)
  
  ```bash
  RESEND_API_KEY=re_...
  ```
</ParamField>

<ParamField path="RESEND_WEBHOOK_SECRET" type="string">
  Resend webhook secret for email event verification.
  
  ```bash
  RESEND_WEBHOOK_SECRET=whsec_...
  ```
</ParamField>

<ParamField path="SMTP_HOST" type="string">
  SMTP server hostname (recommended for local development).
  
  **Local development with MailHog:**
  ```bash
  SMTP_HOST=localhost
  ```
</ParamField>

<ParamField path="SMTP_PORT" type="number">
  SMTP server port.
  
  **Default:** `1025` (MailHog)
  
  ```bash
  SMTP_PORT=1025
  ```
</ParamField>

<ParamField path="SMTP_USER" type="string">
  SMTP authentication username.
  
  ```bash
  SMTP_USER=smtpUser
  ```
</ParamField>

<ParamField path="SMTP_PASSWORD" type="string">
  SMTP authentication password.
  
  ```bash
  SMTP_PASSWORD=smtpPassword
  ```
</ParamField>

<ParamField path="UNSUBSCRIBE_TOKEN_SECRET" type="string">
  Secret for email unsubscribe token generation. Falls back to `NEXTAUTH_SECRET` if not set.
  
  ```bash
  UNSUBSCRIBE_TOKEN_SECRET=...
  ```
</ParamField>

## Internal Use Variables

These variables are used internally by Dub.co and are not required for self-hosted installations.

<Warning>
  The following variables are for official Dub.co infrastructure only. You do not need to set these for self-hosted instances.
</Warning>

- `NEXT_PUBLIC_IS_DUB` - Enables Dub.co specific features
- `CRON_SECRET` - Secret for authenticating cron jobs
- `SLACK_CLIENT_ID` / `SLACK_CLIENT_SECRET` / `SLACK_SIGNING_SECRET` - Slack integration
- `DYNADOT_API_KEY` / `DYNADOT_BASE_URL` / `DYNADOT_COUPON` - Domain registration
- `TWITTER_CLIENT_ID` / `TWITTER_CLIENT_SECRET` - Twitter verification
- `TIKTOK_CLIENT_ID` / `TIKTOK_CLIENT_SECRET` - TikTok verification
- `LINKEDIN_CLIENT_ID` / `LINKEDIN_CLIENT_SECRET` - LinkedIn verification
- `YOUTUBE_API_KEY` - YouTube verification
- `SCRAPECREATORS_API_KEY` - Creator scraping service
- `PAYPAL_CLIENT_ID` / `PAYPAL_CLIENT_SECRET` / `PAYPAL_WEBHOOK_ID` - PayPal integration
- `STRIPE_APP_SECRET_KEY_TEST` / `STRIPE_APP_SECRET_KEY` / `STRIPE_APP_SECRET_KEY_SANDBOX` - Stripe app keys
- `FIRECRAWL_API_KEY` - Program lander generation
- `SINGULAR_WEBHOOK_TOKEN` - Singular events
- `HUBSPOT_CLIENT_ID` / `HUBSPOT_CLIENT_SECRET` - HubSpot integration

## Environment File Template

Here's a minimal `.env` template to get started:

```bash
# Application
NEXT_PUBLIC_APP_NAME=Dub
NEXT_PUBLIC_APP_DOMAIN=yourdomain.com
NEXT_PUBLIC_APP_SHORT_DOMAIN=short.yourdomain.com

# Database
DATABASE_URL="mysql://user:password@host:3306/dub"

# Authentication
NEXTAUTH_SECRET=your-super-secret-key-minimum-32-characters
NEXTAUTH_URL=https://yourdomain.com

# Tinybird Analytics
TINYBIRD_API_KEY=p.ey...
TINYBIRD_API_URL=https://api.tinybird.co

# Vercel Domains API
PROJECT_ID_VERCEL=prj_...
TEAM_ID_VERCEL=team_...
AUTH_BEARER_TOKEN=...

# Upstash Redis
UPSTASH_REDIS_REST_URL=https://....upstash.io
UPSTASH_REDIS_REST_TOKEN=...

# Upstash QStash
QSTASH_TOKEN=...
QSTASH_CURRENT_SIGNING_KEY=...
QSTASH_NEXT_SIGNING_KEY=...

# Email (Choose Resend OR SMTP)
RESEND_API_KEY=re_...
# OR for local development:
# SMTP_HOST=localhost
# SMTP_PORT=1025
```

## Validation

To verify your environment configuration:

1. Check that all required variables are set
2. Test database connectivity with `pnpm prisma:studio`
3. Verify Redis connection by starting the dev server
4. Test email delivery by signing up for an account

<Note>
  Missing required environment variables will cause the application to fail at startup with clear error messages indicating which variables are missing.
</Note>
